2. Gestión de Proyectos

La finalidad es clasificar prioridades, no administrar proyectos complejos.

Proyectos

Funcionalidades

* Crear proyecto.
* Editar proyecto.
* Activar / desactivar proyecto.
* Asignar participantes.

Datos principales

* Nombre
* Descripción
* Estado
* Responsable

⸻

Fases de Proyecto

Permite categorizar el trabajo.

Ejemplo

Proyecto: Implementación CRM

Fases:

* Descubrimiento
* Diseño
* Desarrollo
* Pruebas
* Implementación

Funcionalidades

* Crear fase.
* Editar fase.
* Asociar prioridades a fases.

⸻

3. Check-In Semanal

Es la funcionalidad principal del producto.

Objetivo

Registrar compromisos para la semana.

Flujo

1. Selecciona proyectos.
2. Selecciona fases.
3. Define prioridades.
4. Define tareas.
5. Selecciona prioridades pendientes.
6. Registra riesgos previstos.
7. Envía Check-In.

Gestión de Prioridades

Funcionalidades

* Crear prioridad.
* Editar prioridad.
* Duplicar prioridad.
* Reutilizar prioridad de semana anterior.
* Asociar a proyecto.
* Asociar a fase.

Datos

* Nombre
* Descripción
* Proyecto
* Fase
* Nivel de prioridad
* Estado

Gestión de Tareas

Funcionalidades

* Crear tarea.
* Editar tarea.
* Marcar completada.
* Mover a siguiente semana.

Datos

* Nombre
* Descripción
* Estado

⸻

⸻

5. Check-Out Semanal

Objetivo

Registrar resultados obtenidos.

Flujo

1. Revisa prioridades.
2. Marca completadas.
3. Marca tareas completadas.
4. Registra comentarios.
5. Registra aprendizajes.
6. Registra bloqueadores.
7. Define elementos que continúan.
8. Envía Check-Out.

Continuidad Automática

Funcionalidades

* Arrastrar prioridades pendientes.
* Arrastrar tareas pendientes.
* Crear borrador del siguiente Check-In.

Esta funcionalidad reduce significativamente la fricción.

8. Commitment Reliability Score (CRS)

Funcionalidad Insignia

Es el principal diferenciador del producto.

Objetivo

Medir confiabilidad de cumplimiento.

Variables consideradas

* Prioridades comprometidas.
* Prioridades completadas.
* Tareas comprometidas.
* Tareas completadas.
* Prioridades arrastradas.
* Consistencia histórica.

Interpretación

CRS	Interpretación
90-100	Cumplimiento excelente
80-89	Cumplimiento sólido
70-79	Atención preventiva
Menos de 70	Requiere seguimiento

10. Asistente de IA (Complementario)

Resumen Semanal Automático

Generar:

Esta semana el equipo completó el 87% de sus compromisos. Se detectaron 3 prioridades en riesgo y 2 bloqueadores recurrentes.

Preparación de 1:1

Generar automáticamente:

* Prioridades completadas.
* Pendientes.
* Riesgos.
* Variación del CRS.

Insights para Managers

Ejemplos:

* Juan ha trasladado prioridades durante 4 semanas consecutivas.
* Ana mantiene un CRS superior a 90 durante 8 semanas.
* El proyecto CRM concentra el 60% de los bloqueadores del equipo.
