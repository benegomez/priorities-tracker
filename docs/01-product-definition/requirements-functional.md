Functional Requirements

Introduction

This document defines the functional requirements for Priorities Tracker.

Functional requirements describe the capabilities that the platform must provide to support weekly commitment management, execution visibility, and commitment reliability measurement.

⸻

User and Organization Management

FR-001 User Management

The system shall allow administrators to create users.

FR-002 User Maintenance

The system shall allow administrators to update user information.

FR-003 User Activation

The system shall allow administrators to activate and deactivate users.

FR-004 Role Assignment

The system shall allow assignment of the following roles:

* Administrator
* Manager
* Employee

FR-005 Team Assignment

The system shall allow users to be assigned to teams.

FR-006 Manager Assignment

The system shall allow managers to be assigned to employees.

⸻

Team Management

FR-007 Team Creation

The system shall allow administrators to create teams.

FR-008 Team Maintenance

The system shall allow administrators to edit teams.

FR-009 Team Membership

The system shall allow administrators to assign employees to teams.

⸻

Project Management

FR-010 Project Creation

The system shall allow administrators to create projects.

FR-011 Project Maintenance

The system shall allow administrators to edit projects.

FR-012 Project Activation

The system shall allow projects to be activated and deactivated.

FR-013 Phase Management

The system shall allow project phases to be created and maintained.

⸻

Weekly Planning

FR-014 Check-In Creation

Employees shall be able to create weekly Check-Ins.

FR-015 Priority Creation

Employees shall be able to create priorities.

FR-016 Task Creation

Employees shall be able to create tasks associated with priorities.

FR-017 Priority Reuse

Employees shall be able to reuse pending priorities from previous weeks.

FR-018 Risk Registration

Employees shall be able to register anticipated risks.

⸻

Weekly Execution

FR-019 Progress Updates

Employees shall be able to update progress during the week.

FR-020 Blocker Management

Employees shall be able to register blockers.

FR-021 Risk Updates

Employees shall be able to update risks.

⸻

Weekly Completion

FR-022 Check-Out Creation

Employees shall be able to complete weekly Check-Outs.

FR-023 Completion Tracking

Employees shall be able to mark priorities and tasks as completed.

FR-024 Continuity Management

The system shall support carry-over of unfinished priorities and tasks.

⸻

Dashboards

FR-025 Employee Dashboard

The system shall provide an employee dashboard.

FR-026 Team Dashboard

The system shall provide a manager dashboard.

FR-027 Weekly View

Managers shall be able to review weekly activity across the team.

FR-028 Individual View

Managers shall be able to review employee-level information.

⸻

Reporting

FR-029 Individual Reports

The system shall generate employee reports.

FR-030 Team Reports

The system shall generate team reports.

FR-031 Project Reports

The system shall generate project reports.

⸻

CRS

FR-032 CRS Calculation

The system shall calculate Commitment Reliability Scores.

FR-033 CRS History

The system shall maintain historical CRS information.

⸻

Artificial Intelligence

FR-034 Weekly Summary Generation

The system shall generate AI-powered weekly summaries.

FR-035 Insight Generation

Future releases may provide AI-generated insights and recommendations.
