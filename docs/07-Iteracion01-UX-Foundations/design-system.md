# Design System

## Purpose

Define the visual language and design principles for Priorities Tracker.

## Design Philosophy

Professional SaaS platform focused on managers, accountability and execution visibility.

## Design Principles

### Clarity First
Current status, risks and priorities must be immediately visible.

### Data Over Decoration
Information takes precedence over visual effects.

### Consistency
Reuse patterns across all screens.

### Accessibility
Components must support accessibility best practices.

### Mobile Compatibility
Critical workflows must function correctly on mobile devices.

## Design Stack

- Next.js 15
- TailwindCSS
- shadcn/ui
- Lucide Icons
- Recharts
