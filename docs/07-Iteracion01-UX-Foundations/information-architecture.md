# Information Architecture

## Purpose

Define the logical structure of the Priorities Tracker frontend.

## Application Structure

The application is divided into four major areas:

- Public
- Employee
- Manager
- Administration

## Public Area

Modules:
- Authentication
- Password Recovery

Screens:
- Login
- Forgot Password
- Reset Password

## Employee Area

Modules:
- Dashboard
- Check-In
- Check-Out
- Priorities
- Tasks
- History
- Profile

## Manager Area

Modules:
- Dashboard
- Team
- Reports
- CRS Analytics
- AI Insights

## Administration Area

Modules:
- Users
- Teams
- Projects
- Phases
- Settings

## Cross-Cutting Modules

- Notifications
- Search
- Audit Trail
- Help
