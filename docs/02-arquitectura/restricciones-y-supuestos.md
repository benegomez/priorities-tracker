# Restricciones y Supuestos

## Restricciones Tecnológicas

### Frontend

- React
- Next.js

### Backend

- Python
- FastAPI

### Persistencia

- PostgreSQL

### Cache

- Redis

### Contenedores

- Docker Compose

## Restricciones Organizacionales

- Equipo pequeño.
- Presupuesto controlado.
- MVP orientado a validación.

## Restricciones de Producto

- IA complementaria.
- No gestor de proyectos tradicional.
- Enfoque en compromisos semanales.

## Supuestos

### Usuarios

- Menos de 1000 usuarios inicialmente.

### Equipos

- Entre 5 y 50 personas.

### Carga

- Crecimiento gradual.

## Riesgos Asociados

### Riesgo

Adopción más rápida de lo esperado.

Mitigación:

Preparar migración a Kubernetes.

### Riesgo

Dependencia excesiva de proveedor IA.

Mitigación:

Implementar AI Gateway.

## Decisiones Congeladas

- No microservicios para MVP.
- No Kubernetes para MVP.
- No Event Driven Architecture para MVP.
- No múltiples bases de datos.
